# _PLAN

Planning artifacts only.

## What Goes Here

- `010_PRD.md` for the high-level product direction
- feature folders for scoped planning work
- requirement docs: `010_<feature>.REQ.md`
- plan docs: `020_<feature>.PLAN.md`

## Workflow

1. Update the PRD when direction changes.
2. Create a feature requirement doc with `f-planner req "<Feature>"`.
3. Create a feature plan doc with `f-planner plan "<Feature>"`.
4. Once the plan is accepted, move execution into `_TICKETS/`.
