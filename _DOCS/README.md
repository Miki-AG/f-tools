# _DOCS

Post-implementation documentation only.

## What Goes Here

- usage guides
- operational notes
- behavior references
- docs generated with `f-planner doc "<Topic>"`

Do not use `_DOCS` for planning drafts or execution tracking. That belongs in `_PLAN` and `_TICKETS`.

## Workflow

1. Implement the behavior.
2. Create or update a doc in `_DOCS/`.
3. Keep examples and naming aligned with the actual CLI and file layout.
